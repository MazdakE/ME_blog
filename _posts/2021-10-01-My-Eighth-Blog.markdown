# Introduction 
In this blog post we will talk about how to work with Azure Storage Service Blob, how to create a blob container and uploading blobs in our blob containers. I'll first create a C# Console Application, setting up a connection with my Azure Storage Service Blob which has our blob container with our files/blobs and a simple Web Application where we get our images from a specific blob container so that we can view the images inside our Azure application where we will publish our Web App. The main target of this exercise is to upload an image locally to our blob container in our blob storage and get the URL of that image so that we can view it from our web browser or display the images in our web app. I have included some methods with different functionality in my Console App so you can be able to store multiple blobs inside a blob container, you can also create a new blob container, select a specific blob container and also get a reminder that the blob container that you are creating already exists or not. 

So we create a Storage Account in Azure portal, then we create a Console Application in C# and in our application we are going to make some logics so it's possible to create new containers inside our storage account an push images to our container. It's also good to have some error handling in order for our application to work properly. Lastly we will create a Web App in order to view the images from web.

The application in it self is not that user friendly though because I wanted to make it work properly and lack of time, but this exercise main focus was to make it work.

Before I started this assignemnt I had to download package Azure.Storage.Blobs.

First thing we do in this Console Application is that we create a connection string variable which connects us to our storage account so that we get access to blob storage. The reason we did this was to configuring the connection string with an environment variable `setx AZURE_STORAGE_CONNECTION_STRING "<yourconnectionstring>"` which we wrote in our PowerShell. We confirmed that is worked by displaying the variable with `Console.WriteLine`. 

Next step was to connect our BlobStorageAccount by creating an instance of that class and adding the connection string in its constructor, like this ` BlobServiceClient blobServiceClient = new BlobServiceClient(connectionString);`. BlobStorageAccount gives us the ability to manipulate our Azure Storage Service and blob containers. This is necessary to have in order to work with our blobs.  

After that we display all the available containers that exists. We also created a switch-case in order to select a option depending on if we want to create a blob container, select an existing blob container and upload a blob inside of it (not an necessary option in this case) and upload an image inside an existing blob container. 

The methods are given in each case depending on with option we choose to work with. The methods are self explanatory by there names.
